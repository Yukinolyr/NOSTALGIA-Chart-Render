# NOSTALGIA 谱面 XML 格式完全解析

> 基于对大量谱面 XML（含 m_l0061_felys、m_t0077_atm09_dancer、m_n0001_saturated_world、m_c0002_chopin_nocturn2 等）的遍历分析。

---

## 一、根结构

```xml
<?xml version="1.0"?>
<music_score>
  <header>...    </header>      <!-- 谱面元信息 -->
  <note_data>...  </note_data>    <!-- 音符数据 -->
  <event_data>... </event_data>   <!-- 事件数据（BPM 等）-->
  <beat_data>...  </beat_data>    <!-- 节拍标记 -->
  <track_info>... </track_info>   <!-- 音色轨道映射 -->
  <velocity_zone_data>...</velocity_zone_data> <!-- 力度区域 -->
</music_score>
```

已发现的 6 个顶级节点：`header`、`note_data`、`event_data`、`beat_data`、`track_info`、`velocity_zone_data`

---

## 二、header — 谱面元信息

```xml
<header>
  <max_scale __type="s32">85</max_scale>
  <min_scale __type="s32">8</min_scale>
  <file_version __type="s16">1</file_version>
  <first_bpm __type="s64">15000000</first_bpm>
  <music_finish_time_msec __type="s32">116800</music_finish_time_msec>
</header>
```

| 字段 | 类型 | 含义 |
|------|------|------|
| `max_scale` | s32 | 谱面使用的最高钢琴键编号 |
| `min_scale` | s32 | 谱面使用的最低钢琴键编号 |
| `file_version` | s16 | 固定为 `1` |
| `first_bpm` | s64 | 初始 BPM，实际值 = 存储值 / 100000 |
| `music_finish_time_msec` | s32 | 歌曲结束时间（毫秒）|

## 三、note_data / note — 音符数据

```xml
<note>
  <index __type="s32">937</index>
  <start_timing_msec __type="s32">28400</start_timing_msec>
  <end_timing_msec __type="s32">28525</end_timing_msec>
  <gate_time_msec __type="s32">125</gate_time_msec>
  <scale_piano __type="u8">30</scale_piano>
  <min_key_index __type="s32">8</min_key_index>
  <max_key_index __type="s32">10</max_key_index>
  <note_type __type="s32">4</note_type>
  <hand __type="s32">1</hand>
  <key_kind __type="s32">0</key_kind>
  <param1 __type="s32">-1</param1>
  <param2 __type="s32">938</param2>
  <param3 __type="s32">0</param3>
  <sub_note_data>
    <sub_note>...</sub_note>
  </sub_note_data>
</note>
```

### 3.1 note 字段详解

| 字段 | 类型 | 含义 |
|------|------|------|
| `index` | s32 | 全局唯一序号，从 1 开始。Glissando 链通过此 index 在 `param1`/`param2` 中引用 |
| `start_timing_msec` | s32 | 音符开始时间（毫秒）|
| `end_timing_msec` | s32 | 音符结束时间（毫秒）|
| `gate_time_msec` | s32 | 音符持续时间（毫秒）|
| `scale_piano` | u8 | 中央音高/基准音高（0~87 对应 88 键）|
| `min_key_index` | s32 | 音符占据的最左琴键索引（0-based）|
| `max_key_index` | s32 | 音符占据的最右琴键索引（0-based，渲染时按 `max-min` 计算宽度）|
| `note_type` | s32 | 音符类型，见下表 |
| `hand` | s32 | `0`=右手(红)、`1`=左手(蓝)、`2`=隐藏/系统音符 |
| `key_kind` | s32 | **所有谱面恒为 `0`，用途未知** |
| `param1` | s32 | Glissando/Chain 时 = 前一节点 index；链头 = `-1` |
| `param2` | s32 | Glissando/Chain 时 = 后一节点 index；链尾 = `-1` |
| `param3` | s32 | **所有谱面恒为 `0`，用途未知** |

### 3.2 note_type 类型表

| 值 | 内部名 | 说明 | 纹理基础名 |
|----|--------|------|-----------|
| `0` | Normal | 普通单点音符（白色）| `white` |
| `2` | Long | 长押音符（白色 sustained）| `white` |
| `4` | Glissando | 滑音链节点（有头有尾，通过 param1/param2 链接）| `glissando` |
| `8` | Normal-sfx | 单点音符（灰色，通常 hand=2）| `white`（渲染时 override 为 0）|
| `10` | Special | 特殊长押（金色 sustained）| `white_forte` |
| `12` | Chain | 滑音链节点（灰色版，功能同 type 4）| 渲染时 override 为 `glissando` |
| `64` | Trill | 颤音（橙色 sustained）| `trill_forte` |

**已确认**：实际游戏中仅出现以上 7 种。`hand=2` 的隐藏音符必须过滤，不参与渲染。

### 3.3 sub_note_data / sub_note — 音源触发信息

每个 `note` 内含 1~N 个 `sub_note`，用于映射到 general_key 钢琴采样：

```xml
<sub_note>
  <start_timing_msec __type="s32">9200</start_timing_msec>
  <end_timing_msec __type="s32">11200</end_timing_msec>
  <scale_piano __type="u8">63</scale_piano>
  <velocity __type="u8">63</velocity>
  <track_index __type="s32">1</track_index>
</sub_note>
```

| 字段 | 类型 | 含义 |
|------|------|------|
| `start_timing_msec` | s32 | 采样触发开始时间 |
| `end_timing_msec` | s32 | 采样触发结束时间 |
| `scale_piano` | u8 | 触发的具体琴键（0~87）|
| `velocity` | u8 | MIDI 力度值（0~127）|
| `track_index` | s32 | 映射到 `track_info` 中的音色轨道编号 |

**特殊情况**：Trill (type 64) 通常包含大量 sub_note（6~10+ 个），用于模拟快速交替音；其他类型通常只有 1 个。

## 四、event_data / event — 事件数据

```xml
<event>
  <index __type="s32">0</index>
  <start_timing_msec __type="s32">0</start_timing_msec>
  <type __type="s32">0</type>
  <value __type="s64">15000000</value>
</event>
```

| type | 含义 | 值解析 |
|------|------|--------|
| `0` | **BPM 变更** | 实际 BPM = value / 100000 |
| `1`~`8` | **未知** | 只在谱面开头出现，可能与滚动速度/判定线/视觉效果相关 |

已观测到的 type 1~8 初始值：

| type | value | 推测 |
|------|-------|------|
| 1 | 120 | 基准下落速度 (%) |
| 2 | 11 | 未知 |
| 3 | 24 | 判定线 Y 坐标相关？|
| 4 | 20 | 未知 |
| 5 | 24 | 与 type 3 相同 |
| 6 | 120 | 与 type 1 相同 |
| 7 | 0 | 布尔开关？|
| 8 | 80 | 未知 |

## 五、beat_data / beat — 节拍标记

```xml
<beat>
  <index __type="s32">0</index>
  <start_timing_msec __type="s32">0</start_timing_msec>
</beat>
```

| 字段 | 类型 | 含义 |
|------|------|------|
| `index` | s32 | 节拍序号（从 0 开始）|
| `start_timing_msec` | s32 | 该节拍出现的时间（毫秒）|

---

## 六、track_info / track — 音色轨道映射

```xml
<track_info>
  <track>
    <index __type="s32">1</index>
    <name __type="str">key_apiano5</name>
  </track>
</track_info>
```

| 字段 | 类型 | 含义 |
|------|------|------|
| `index` | s32 | 轨道编号，被 `sub_note.track_index` 引用 |
| `name` | str | 对应的 general_key XWB 文件名（不含扩展名）|

---

## 七、velocity_zone_data / velocity_zone — 力度区域

```xml
<velocity_zone>
  <index __type="s32">0</index>
  <start_timing_msec __type="s32">1200</start_timing_msec>
  <end_timing_msec __type="s32">14600</end_timing_msec>
  <velocity_type __type="s32">1</velocity_type>
</velocity_zone>
```

| 字段 | 类型 | 含义 |
|------|------|------|
| `index` | s32 | 区域序号 |
| `start_timing_msec` | s32 | 区域开始时间 |
| `end_timing_msec` | s32 | 区域结束时间 |
| `velocity_type` | s32 | `0` 或 `1`。当前渲染约定：`0`=轻段，`1`=重段 |

> **渲染约定**：轻段使用浅蓝色背景，重段使用浅黄色背景；如后续实机核对发现 0/1 相反，只需交换渲染颜色映射。

## 八、Glissando / Chain 链解析规则

`note_type=4` (Glissando) 和 `note_type=12` (Chain) 通过 `param1`/`param2` 构成单向链表：

- **链头**：`param1 == -1`，`param2 == 下一节点 index`
- **中间节点**：`param1 == 上一节点 index`，`param2 == 下一节点 index`
- **链尾**：`param1 == 上一节点 index`，`param2 == -1`

**注意**：链中的节点 `hand` 值可能不同，但通常同色。`hand=2` 的节点不参与链的构建。

---

## 九、文件名约定

谱面 XML 文件名格式：
```
{basename}_{diff}.xml
```

难度后缀：
| 后缀 | 难度 |
|------|------|
| `_00normal` | Normal |
| `_01hard` | Hard |
| `_02extreme` | Expert |
| `_03real` | Real |

部分歌曲还包含 `_pre`（预览谱面）或 `_april`（愚人节版本）。

## 十、未知/待确认项汇总

| 项目 | 状态 |
|------|------|
| `key_kind` 字段 | 恒为 `0`，用途完全未知 |
| `param3` 字段 | 恒为 `0`，用途完全未知 |
| `event_data` 中 type `1`~`8` | 可能与下落速度/判定线/视觉效果相关 |
| `velocity_zone_data.velocity_type` 的 `0`/`1` | 当前按 `0`=轻段、`1`=重段渲染，仍需实机核对 |
| `note_type=8` 与 `note_type=0` 在游戏内的具体差异 | 外观相同，但 type 8 更多用于 hand=2 |
| `note_type=12` (Chain) 与 `note_type=4` (Glissando) 的具体差异 | 渲染上均作为滑音链，但 Chain 使用灰色纹理 |
